朋友演示版

只想看效果，不填 API Key 也能跑演示 Excel

1. 打开链接

朋友打开你发的网址，就能看到这个页面。

2. 不填 Key 也可以

不填 API Key 时，只跑演示数据，不会产生费用。

3. 点“先估算费用”

先看如果真实跑大概要调用几次、多少钱。

4. 点“确认运行”

生成演示 Excel。真实采集时再填自己的 TikHub API Key。

真实 API 流程

照着这 5 步走，第一次真实测试不会乱花钱

  1. 1
    去 TikHub 用户中心复制 API Key

    它像门钥匙。没有钥匙时，本工具只会跑演示数据，不会扣费。

  2. 2
    点右上角 API Key，粘贴并保存到当前浏览器

    每个人只保存自己的 Key。朋友之间不会互相覆盖。

  3. 3
    点“测试连接”，确认钥匙能用

    只检查 Key，不会开始抓小红书数据。

  4. 4
    填关键词和安全上限，先估算费用

    看到预计调用次数和预计费用，再决定要不要跑。

  5. 5
    点“确认运行”，等待 Excel

    第一轮建议：关键词“墨西哥海外仓”，结果数量 3，最高费用 $0.05。

任务怎么自动关联 API

你点的是业务任务，程序背后自动选接口

市场素材搜索

先调用“搜索笔记”，再按结果自动调用“图文/视频笔记详情”。

竞品账号研究

自动调用“搜索用户 → 用户信息 → 用户发布笔记”。

评论需求分析

自动调用“搜索笔记 → 笔记评论”，后续再加二级评论。

话题趋势研究

先用“搜索笔记”找线索；如果结果里有话题 ID，再调用话题接口。

商品与用户痛点研究

自动调用“搜索商品 → 商品详情 → 商品评论”。

图片 / 群聊搜索

分别自动调用“搜索图片”和“搜索群聊”。

你截图里“分享链接为空”的原因

不是你 API Key 的问题，也不是你任务点错了；是之前 Excel 只读取了一个链接字段。现在已经修成多字段查找，并且用笔记 ID 兜底生成链接。

任务设置

选任务，填关键词

想要结果数量

你要几条“成品结果”。填 3，就是最多要 3 篇笔记或 3 个账号。

最多调用

程序问 TikHub 一次就算 1 次调用。搜 1 次、查 3 篇详情，大概就是 4 次。

最多花费

本次最多允许花多少钱。比如填 0.05，就是超过 5 美分马上停。

测试模式

默认最多 30 次调用。真实测试也建议先开着。
