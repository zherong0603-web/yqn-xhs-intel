# YQN 小红书 API 能力说明

更新时间：2026-06-15

数据来源：
- TikHub 官方文档：https://docs.tikhub.io/
- TikHub OpenAPI：https://api.tikhub.io/openapi.json
- TikHub 端点查价接口：`/api/v1/tikhub/user/get_all_endpoints_info`

重要提醒：
- 当前 Xiaohongshu-App-V2-API 每个接口查到的单价都是 **0.01 美元/次**。
- 价格以后可能变化，所以程序运行前会重新查价，再给你看预计费用。
- 你没有点“确认运行”之前，程序不会开始正式调用业务接口。
- API Key 只保存在本机，不写进代码，不上传 GitHub。

## 一句话理解

API 就像 20 个不同的小按钮。每个按钮问小红书拿一种东西：有的拿笔记，有的拿评论，有的拿用户，有的拿商品，有的拿话题。YQN 助手会先看你的任务，再决定按哪几个按钮，不会每次把 20 个按钮全按一遍。

## 1. 图文笔记详情

能做什么：
- 把一篇图文笔记完整拿回来，比如标题、正文、图片、作者、点赞、收藏、评论数。

YQN 什么时候用：
- 看到一篇可能值得模仿的素材时，用它把内容细节拿全。
- 做“市场素材搜索”时，搜索到笔记后，再挑少量笔记拿详情。

需要输入什么：
- `note_id` 笔记 ID，或者 `share_text` 分享链接。

能返回什么：
- 笔记正文、图片、作者信息、互动数据、发布时间等。

是否值得优先使用：
- 值得，但不要一上来全量用。先搜索，再只查前几篇高价值笔记。

可能调用多少次：
- 小测试：最多 3 次。
- 正式小任务：10 到 50 次。

当前接口单价：
- $0.01/次。

## 3. 笔记评论

能做什么：
- 拿到某篇笔记下面的一页评论。

YQN 什么时候用：
- 做“评论需求分析”，看用户在问什么、抱怨什么、想买什么。

需要输入什么：
- `note_id` 或 `share_text`。
- 可选：排序方式，比如按最新或按点赞。
- 翻页时还需要上一次返回的 `cursor`、`index`、`pageArea`。

能返回什么：
- 评论内容、评论用户、点赞数、是否有回复、下一页线索。

是否值得优先使用：
- 非常值得。评论是找需求和痛点的核心。

可能调用多少次：
- 小测试：0 到 3 次。
- 正式小任务：每篇笔记 1 到 5 次，受费用上限控制。

当前接口单价：
- $0.01/次。

## 4. 二级评论

能做什么：
- 拿到某条评论下面的回复。

YQN 什么时候用：
- 某条评论下有很多人讨论时，用它看更细的需求。
- 比如有人问“墨西哥清关多久”，下面一堆回复，就值得看二级评论。

需要输入什么：
- `note_id` 或 `share_text`。
- `comment_id` 父评论 ID。
- 翻页时需要 `cursor` 和 `index`。

能返回什么：
- 回复内容、回复用户、下一页线索。

是否值得优先使用：
- 不作为第一优先。先看一级评论，发现重点评论后再用。

可能调用多少次：
- 小测试：0 到 1 次。
- 正式小任务：只对重点评论调用，通常 5 到 20 次内。

当前接口单价：
- $0.01/次。

## 第一阶段按钮怎么选择接口

### 市场素材搜索

默认会用：
- 搜索笔记。
- 只对目标数量内的笔记调用图文笔记详情或视频笔记详情。

不会默认用：
- 评论、商品、群聊、创作者灵感。

### 竞品账号研究

默认会用：
- 搜索用户。
- 用户信息。
- 用户发布笔记。

可选再用：
- 用户收藏笔记。

### 评论需求分析

默认会用：
- 搜索笔记。
- 笔记评论。

可选再用：
- 二级评论。

### 话题趋势研究

默认会用：
- 搜索笔记，先找相关话题线索。
- 如果拿到话题 ID，再用话题详情和话题笔记。

可选再用：
- 创作者热点灵感。

### 商品与用户痛点研究

默认会用：
- 搜索商品。
- 商品详情。
- 商品评论总览。
- 商品评论列表。

可选再用：
- 商品推荐。

### 图片补充搜索

默认会用：
- 搜索图片。

### 群聊搜索

默认会用：
- 搜索群聊。

## 费用保护规则

程序必须做到：
- 测试模式默认最多 30 次调用。
- 运行前显示预计调用次数和预计费用。
- 用户点击“确认运行”后才开始。
- 可以设置本次最大调用次数。
- 可以设置本次最高费用。
- 达到上限马上停止。
- 不做自动充值。
- 已经成功拿过的数据存在本机缓存里，下次不会重复付费调用。
