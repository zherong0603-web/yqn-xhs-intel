# 真实连接 TikHub 教程

更新时间：2026-06-15

这份教程只讲怎么点，不讲技术原理。

## 你要准备什么

只需要一样东西：

- TikHub API Key

你可以把它理解成“门钥匙”。没有这把钥匙，运去哪·小红书市场情报助手只能跑演示数据；有了钥匙，才能真实去 TikHub 调小红书接口。

## 第 1 步：打开运去哪助手

双击：

`启动YQN小红书市场情报助手.command`

如果页面没有自动打开，就在浏览器打开：

`http://127.0.0.1:8765`

## 第 2 步：填写 API Key

在页面右上角点：

`API Key`

把 TikHub API Key 粘贴到：

`TikHub API Key`

然后点：

`保存到当前浏览器`

注意：

- API Key 只保存在当前浏览器。
- 不会写进代码。
- 不会上传 GitHub。
- 不会写进服务器配置文件。

## 第 3 步：测试连接

还是在 API Key 弹窗里，点：

`测试连接`

如果看到“连接成功”，说明钥匙能用。

如果失败，通常是：

- API Key 少复制了一段。
- API Key 前后多了空格。
- TikHub 账号余额或权限有问题。

## 第 4 步：做第一次真实测试

推荐这样填：

- 关键词：墨西哥海外仓
- 想要结果数量：3
- 测试模式：打开
- 安全上限：最多调用：30
- 安全上限：最多花费：0.05

然后点：

`市场素材搜索`

再点：

`先估算费用`

看清楚预计费用后，最后点：

`确认运行`

## 任务是不是自动关联 API

是的。

你不用自己选 API，也不用看英文接口名。

你点的是业务任务，程序背后会自动选接口：

- 市场素材搜索：先“搜索笔记”，再查“图文/视频笔记详情”。
- 竞品账号研究：先“搜索用户”，再查“用户信息”和“用户发布笔记”。
- 评论需求分析：先“搜索笔记”，再查“笔记评论”。
- 话题趋势研究：先“搜索笔记”找线索，如果有话题 ID，再查“话题详情”和“话题笔记”。
- 商品与用户痛点研究：先“搜索商品”，再查“商品详情”和“商品评论”。
- 图片补充搜索：查“搜索图片”。
- 群聊搜索：查“搜索群聊”。

## 第 5 步：打开 Excel

跑完后，右侧会出现：

`点击下载 / 打开 Excel`

也可以去这个文件夹找：

`YQN_XHS_Intel/outputs/`

## 本次真实测试会调用什么

如果选择“市场素材搜索”，数量填 3，程序通常会调用：

- 搜索笔记：1 次
- 笔记详情：最多 3 次

当前查到的接口单价是：

- $0.01/次

所以预计最多：

- 4 次
- $0.04

如果你把最高费用设成 $0.05，就算哪里异常，超过保护线也会停止。

## 三个容易混的词

### 想要结果数量

意思是：你想要几条“成品结果”。

例子：

- 填 3，就是最多要 3 篇笔记。
- 填 10，就是最多要 10 篇笔记。

### 安全上限：最多调用

意思是：程序最多可以问 TikHub 几次。

你可以把它理解成“保险丝”。

例子：

- 搜索一次笔记，算 1 次调用。
- 查 3 篇笔记详情，通常再算 3 次调用。
- 所以拿 3 篇笔记，预计大概是 4 次调用。

### 安全上限：最多花费

意思是：本次最多允许花多少钱。

你可以把它理解成“钱包刹车”。

例子：

- 当前小红书 App V2 接口大多是 $0.01/次。
- 填 0.05，就是本次最多花 5 美分。
- 如果程序发现再调用一次会超过 0.05，就会立刻停。

## Excel 里“分享链接”为空是什么意思

如果你看到旧 Excel 里“分享链接”为空，不代表 API Key 错了，也不代表任务错了。

原因是：TikHub 不同接口返回链接的位置不一样，旧版只看了一个位置。

新版已经修正：

- 如果详情接口返回真实分享链接，就优先写真实链接。
- 如果搜索结果没有直接给链接，就用笔记 ID 自动生成一个可打开的小红书链接。
